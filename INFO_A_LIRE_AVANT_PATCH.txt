## Objectif du plugin

Design Studio est un plugin Jeedom séparé de Post-it Design.

But : créer une interface moderne de travail sur les Designs Jeedom :
- page Studio dédiée ;
- Design chargé dans une iframe ;
- dock bas ;
- panneau latéral ;
- scan des objets ;
- sélection au clic gauche ;
- multi-sélection ;
- alignement ;
- déplacement visuel ;
- sauvegarde des positions dans les lignes plan Jeedom.

Le plugin ne doit PAS devenir une simple collection de widgets Jeedom.

## Règles importantes

- Ne pas modifier le plugin postitdesign.
- Ne pas modifier PROD.
- Ne pas toucher MQTT.
- Ne pas redémarrer Apache.
- Ne pas patcher le core Jeedom.
- Ne pas injecter de code dans footer.php ou autre fichier core.
- Ne pas utiliser /var/www/html/core/js/custom.js comme loader durable.
- Toute fonction doit rester isolée dans /var/www/html/plugins/designstudio.
- Toute étape doit avoir backup + commit Git.
- Avant tag, ce fichier doit être mis à jour.

## Architecture retenue

Le bon modèle retenu est :

1. Page admin Design Studio :
   /index.php?v=d&m=designstudio&p=designstudio

2. Page Studio dédiée :
   /index.php?v=d&m=designstudio&p=studio&plan_id=XX

3. Le Design Jeedom est chargé dans une iframe.

4. Le Studio lit les objets du Design dans l’iframe :
   - .eqLogic-widget
   - .cmd-widget
   - .postitdesign-widget

5. Les modifications sont d’abord visuelles, puis sauvegardées via AJAX dans les lignes plan Jeedom.

## Procédure de test minimale avant tag

1. Ouvrir Design Studio.
2. Ouvrir un Design dans Studio.
3. Cliquer Reload.
4. Cliquer Scan.
5. Sélectionner un objet au clic gauche.
6. Déplacer avec les flèches.
7. Enregistrer position.
8. Reload.
9. Vérifier que l’objet reste à sa nouvelle position.
10. Activer multi-sélection.
11. Sélectionner au moins deux objets.
12. Tester aligner gauche ou haut.
13. Enregistrer sélection.
14. Reload.
15. Vérifier que l’alignement reste.
